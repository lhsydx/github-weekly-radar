# ChatGPT 每周解读提示词

请读取以下公开数据：

- 最新结构化数据：`https://raw.githubusercontent.com/lhsydx/github-weekly-radar/main/data/latest.json`
- 最新原始榜单：`https://raw.githubusercontent.com/lhsydx/github-weekly-radar/main/reports/latest.md`

基于最新数据生成一份中文《GitHub 开源项目周报》。必须保持两张榜单的原始名次，不对重复项目补位；同时进入两榜的项目标记“⭐双榜上榜”。

报告采用“决策版＋双视角”，结构如下：

1. **本周一句话判断**：用一句话概括本周开源趋势。
2. **三大趋势**：提炼 AI与智能体、医疗数智化、效率与开发工具中的重要变化，不强行保证每个赛道都有趋势。
3. **本周最值得关注的 3 个项目**：说明选择原因。
4. **近30天新项目 Star Top 10**：表格展示名次、项目、赛道、总 Star、语言、关注等级和链接。
5. **7天增长 Top 10**：表格展示名次、项目、赛道、总 Star、7天新增、增长率、关注等级和链接。若数据的 `ranking_mode` 为 `cold_start_daily_average`，明确标注“首周观察榜，按日均 Star 排名，并非真实7天增量”。
6. **项目决策卡**：每个唯一项目只分析一次，包括：
   - 解决什么问题
   - 为什么本周走红
   - 适合谁使用
   - 通用技术或产品价值
   - 对医疗AI、医共体产品、数智化方案或产品团队的启发
   - 采用成本与主要风险
   - 建议：立即体验 / 持续关注 / 暂不关注
7. **给罗鸿森的行动建议**：最多3条，必须具体可执行。

要求：

- 使用项目仓库、README、官方文档作为主要依据；不要根据项目名称臆测。
- 数据字段以 `data/latest.json` 为准。
- 对无法验证的信息明确说明，不虚构功能、客户或商业化情况。
- 全文简洁，优先帮助产品负责人快速判断“是否值得看、是否值得试、能否借鉴”。
- 每个项目名称附 GitHub 链接。
